# Система Взаимодействия (Interaction System)

В игре используется унифицированная система для общения с NPC и взаимодействия с объектами.

## Основной принцип

Взаимодействие происходит, когда игрок нажимает кнопку **Подтвердить** (обычно `Z` или `Enter`), и соблюдены условия:
1.  Игрок находится рядом с объектом.
2.  Игрок "смотрит" на объект (используется `obj_pointMarker`).
3.  Интерфейс не заблокирован (нет открытых меню).

## Как это работает в коде?

Вместо дублирования проверок в каждом объекте, мы используем одну функцию: `scr_interaction_check`.

### `scr_interaction_check(dialogue_file, dialogue_node)`

Эта функция делает всю грязную работу:
1.  Проверяет ввод (`scr_input_pressed("confirm")`).
2.  Проверяет `scr_checkUIBlocking()` (чтобы нельзя было говорить сквозь меню).
3.  Проверяет коллизию маркера игрока (`obj_pointMarker`) с объектом.

**Возвращает**: `true`, если взаимодействие произошло (и диалог запущен), иначе `false`.

## Пример использования

В событии **Step** вашего объекта (например, `obj_bench`):

```gml
// Проверяем взаимодействие. Если успешно - запустится диалог из узла "Blue" файла "testDialogueBlue.yarn"
scr_interaction_check("testDialogueBlue.yarn", "Blue");
```
