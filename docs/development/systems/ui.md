# Система Интерфейса (UI System)

Система UI (User Interface) управляет окнами, меню и блокировкой ввода.

## Глобальный Стек UI (UI Stack)

Чтобы избежать конфликтов, когда игрок двигается с открытым меню, мы используем **Глобальный Стек UI** (`global.ui_stack`).

### Принцип работы
Это список ("стопка") открытых окон.
*   Когда окно открывается -> оно добавляет себя в стек (`push`).
*   Когда окно закрывается -> оно убирает себя из стека (`pop`).
*   Если в стеке есть хоть один объект -> **Игрок заблокирован** (не может ходить, взаимодействовать).

### Функции

#### `scr_ui_push(instance_id)`
Добавляет объект в список блокировки.
```gml
// В событии Create окна
scr_ui_push(id);
```

#### `scr_ui_pop(instance_id)`
Убирает объект из списка.
```gml
// В событии CleanUp или Destroy окна
scr_ui_pop(id);
```

#### `scr_checkUIBlocking(exclude_self)`
Проверяет, заблокирован ли интерфейс.
*   `exclude_self = true`: Проверяет, есть ли *другие* окна (полезно для самого меню, чтобы оно работало, пока остальные спят).
*   `exclude_self = false`: Проверяет, есть ли вообще какие-то окна (для игрока).

```gml
// В коде игрока
if (!scr_checkUIBlocking()) {
    // Можно двигаться
}
```

## Меню и Настройки
Основные объекты меню (`obj_menu`, `obj_settingsManager`, `obj_saveManager`) автоматически используют эту систему. При создании они блокируют ввод, при уничтожении - разблокируют.
