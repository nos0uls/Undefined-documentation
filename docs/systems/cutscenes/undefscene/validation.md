# Undefscene: Проверки перед экспортом

Перед экспортом редактор проверяет катсцену на ошибки. Результаты отображаются в панели **Логи / Предупреждения**.

## Три уровня проблем

| Уровень | Что значит | Блокирует экспорт? |
|---------|-----------|-------------------|
| :octicons-x-circle-16:{ style="color: red" } Error | Критическая ошибка — катсцена не работает | **Да** |
| :octicons-alert-16:{ style="color: orange" } Warn | Важное предупреждение — лучше исправить | Нет |
| :octicons-light-bulb-16:{ style="color: green" } Tip | Рекомендация — можно пропустить | Нет |

## Частые ошибки (Error)

- Рёбра ссылаются на несуществующие ноды — обычно после удаления ноды, но не удаления связи

## Частые предупреждения (Warn)

- **Пустое обязательное поле** — например, Target у Move не заполнен
- **Изолированная нода** — нода не соединена ни с чем (кроме Start/End)
- **Недостижимая нода** — до ноды нельзя добраться от Start
- **Множественные выходы** — обычная нода имеет больше одного исходящего ребра (используйте Branch или Parallel)
- **Dialogue без файла** — не выбран .yarn файл
- **Camera Shake с duration ≤ 0** — тряска не будет работать
- **Follow Path с пустым путём** — нужно хотя бы 2 точки
- **Halt с исходящими связями** — Halt блокирует выполнение, ноды после него не запустятся

## Проверки с загруженным проектом

Если вы открыли `.yyp` проект, включаются дополнительные проверки:

- Существует ли объект/спрайт, указанный в Actor Create?
- Существует ли `.yarn` файл и нода в Dialogue?
- Существует ли спрайт в Animate / Emote?
- Разрешена ли функция в Run Function?

## Что делать с предупреждениями

1. Откройте панель **Логи / Предупреждения**
2. Кликните по предупреждению — редактор подсветит проблемную ноду
3. Заполните пустые поля, удалите лишние связи, добавьте недостающие
4. После исправления предупреждения исчезнут
