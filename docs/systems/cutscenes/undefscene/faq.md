# Undefscene: FAQ и troubleshooting

## Почему редактор сохраняет `.usc.json`, а игра использует `.json`?

`.usc.json` нужен для продолжения работы в редакторе: там есть позиции нод, состояние панелей и другие editor-only данные.

Обычный `.json` — это чистый экспорт для движка.

## Можно ли открыть обратно уже экспортированный engine `.json`?

Да. Редактор поддерживает reverse-import и восстанавливает граф из массива `actions`, если файл соответствует ожидаемому engine-формату.

## Почему предупреждение о потере текущей сцены появляется при `Open Scene`, а не при `Export`?

Потому что именно `Open Scene` заменяет текущую editor scene в памяти. Экспорт создаёт runtime `.json` для игры и сам по себе не должен считаться заменой открытой рабочей сцены.

## Почему `Runtime JSON` открывается не в обычном наборе панелей?

Это advanced/debug-панель. Она вынесена в `Help > Advanced`, чтобы не мешать основному рабочему интерфейсу.

## Что делать, если в `Логи / Предупреждения` есть warning, но экспорт всё равно доступен?

Если warning не блокирует экспорт, всё равно стоит перепроверить:

- пустые поля
- имена ресурсов
- `dialogue` file/node
- условия на ребре

## Где искать логи приложения для баг-репорта?

Используйте `Help -> Copy Log to Clipboard` или файл `undefscene.log` внутри `userData`.

## Почему проект GameMaker иногда восстанавливается сам после перезапуска редактора?

Редактор сохраняет путь к последнему открытому `.yyp` и cache ресурсов между сессиями. Это ускоряет повторный старт и помогает не открывать один и тот же проект вручную каждый раз.

## Можно ли сделать hot-swap JSON прямо в packaged `datafiles` GameMaker без пересборки?

Как правило, нет. Для fast loop безопаснее считать packaged `datafiles` частью билда.

Если нужен reload без rebuild, runtime должен читать JSON как внешний файл с диска, а не рассчитывать только на packaged bundle. В текущем workflow Undefscene это считается отдельной архитектурной задачей, а не поведением по умолчанию.

## Почему `gridSize` не меняет расстояние между новыми нодами?

Потому что сейчас `gridSize` относится к фоновой сетке canvas. Он меняет визуальный шаг сетки, а не алгоритм размещения новых нод.
