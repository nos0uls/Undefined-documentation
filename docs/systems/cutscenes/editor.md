# Редактор Катсцен (Undefscene)

Редактор катсцен (Undefscene) — это официальное desktop-приложение (на базе Electron и React Flow) для визуального создания, редактирования и тестирования катсцен в проекте Undefinedtale-888.

Редактор позволяет отойти от ручного написания JSON-файлов и предоставляет удобный node-based интерфейс, где логика катсцены выстраивается в виде графа.

## Запуск редактора

Редактор находится в репозитории `Undefscene`.

### Режим разработки (Dev)
Для запуска редактора из исходников:
1. Перейдите в папку `editor-app`
2. Установите зависимости: `npm install`
3. Запустите редактор: `npm run dev`

### Сборка (Build)
Для сборки готового приложения:
1. Перейдите в папку `editor-app`
2. Запустите нужный билд-скрипт, например: `npm run build:win` для Windows.
3. Готовый бинарный файл появится в папке `dist`.

## Интерфейс и Управление

Интерфейс редактора разделен на три основные зоны:
1. **Левая панель (Палитра нод и Действия)**: Здесь можно выбрать ноду для добавления на холст, а также выполнить отмену/возврат действий (Undo/Redo).
2. **Центральный холст (Canvas)**: Основная рабочая область, где располагаются ноды и связи между ними.
3. **Правая панель (Инспектор и Интеграция)**: Показывает параметры выбранной ноды/связи, а также содержит инструменты для загрузки GameMaker проекта и экспорта сцены.

### Управление на холсте
* **Создание ноды**: Нажмите Среднюю Кнопку Мыши (СКМ/Колесико) по пустому месту холста. Будет создана нода того типа, который вы предварительно выбрали (или нода по умолчанию). Также можно добавлять ноды кликом из левой панели.
* **Перемещение холста (Pan)**: Удерживайте Правую Кнопку Мыши (ПКМ) и двигайте мышь.
* **Выделение рамкой (Box Select)**: Зажмите Левую Кнопку Мыши (ЛКМ) на пустом месте и потяните.
* **Выделение ноды/связи**: Клик ЛКМ по ноде или связи.
* **Удаление ноды/связи**: Клик ПКМ по ноде или связи.
* **Соединение нод**: Потяните ЛКМ от кружка (handle) одной ноды к кружку другой. Связи всегда идут слева направо (от `out` к `in`).
* **Настройка ожидания (Wait)**: Двойной клик по связи (ребру) открывает в инспекторе (справа) поле для ввода времени ожидания (в секундах) перед выполнением следующей ноды.

## Интеграция с движком GameMaker

Редактор тесно интегрирован с движком игры для обеспечения автокомплита и валидации.

### Открытие проекта (Open Project)
В правой панели в блоке "Project Integration" нажмите **Open Project** и выберите файл `.yyp` вашего проекта Undefinedtale-888.
После успешной загрузки редактор просканирует ресурсы игры и настройки.

### Возможности интеграции
* **Автокомплит**: При редактировании параметров нод (например, имени объекта, спрайта, звука, комнаты или диалоговой ветки Yarn) редактор будет предлагать существующие ресурсы из загруженного проекта.
* **Синхронизация настроек**: Редактор читает файл `datafiles/cutscene_engine_settings.json` из проекта, чтобы получить настройки по умолчанию (например, дефолтный FPS, списки разрешенных функций и условий).
* **Поддержка Yarn**: Редактор сканирует файлы `.yarn` в папке `datafiles` и предлагает названия нод диалогов для автокомплита.

## Импорт, Экспорт и Форматы

Редактор работает с двумя форматами файлов:

1. **Формат редактора (`.usc.json`)**: Это внутренний формат редактора, содержащий не только логику катсцены, но и визуальные данные (позиции нод на холсте, цвета и т.д.). **Этот файл нужен для сохранения прогресса работы в редакторе.**
2. **Формат движка (`.json`)**: Это чистый экспорт катсцены, который "понимает" движок GameMaker (объект `obj_cutscene_manager`). В нём нет визуальной информации, только структура команд (actions).

### Сохранение и Загрузка сцены (Рабочий процесс)
* **Save Scene / Save As...**: Сохраняет текущую сцену со всеми визуальными данными в файл `.usc.json`.
* **Open Scene**: Открывает ранее сохраненный файл `.usc.json` (или `.json`) в редакторе.

*Примечание: Вы можете открыть и обычный `.json` файл движка, но в этом случае редактор автоматически расставит ноды на холсте по сетке, так как в файле нет данных об их координатах.*

### Экспорт для игры (Export)
* **Export for Engine**: Генерирует чистый `.json` файл с массивом `actions`, готовый для использования в игре.
Сохраняйте этот файл напрямую в папку `datafiles` вашего проекта GameMaker.

## Ручное редактирование

Хотя редактор предоставляет удобный интерфейс, вы по-прежнему можете редактировать `.json` файлы катсцен вручную в любом текстовом редакторе (например, в VS Code), если это необходимо.
Движок ожидает массив объектов (actions), где каждый объект имеет тип (`type`) и параметры. Подробнее о структуре команд движка читайте в разделе [API Катсцен](api.md).

## Поддерживаемые ноды (Типы команд)

Редактор поддерживает все основные команды движка:

* `start` / `end`: Начало и конец катсцены (технические ноды).
* `actor_create` / `actor_destroy`: Управление актерами (создание, удаление).
* `move`: Перемещение актера в точку.
* `follow_path`: Движение актера по пути (Path).
* `animate`: Воспроизведение анимации (спрайта) актером.
* `dialogue`: Запуск диалога (Yarn Spinner).
* `camera_track` / `camera_pan`: Управление камерой.
* `set_position`, `set_depth`, `set_facing`: Моментальное изменение свойств актера.
* `run_function`: Вызов кастомной GML функции (белый список в `cutscene_engine_settings.json`).
* `parallel_start`: Запуск нескольких веток действий параллельно.
* `branch`: Ветвление логики по условию.

Более подробную информацию о параметрах каждой команды смотрите в разделе [API Катсцен](api.md).
