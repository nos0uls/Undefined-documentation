# Редактор Катсцен (Undefscene)

Редактор катсцен (Undefscene) — это официальное desktop-приложение (на базе Electron и React Flow) для визуального создания, редактирования и тестирования катсцен в проекте Undefinedtale-888.

Редактор позволяет отойти от ручного написания JSON-файлов и предоставляет удобный node-based интерфейс, где логика катсцены выстраивается в виде графа.

## Запуск редактора

Редактор находится в репозитории `Undefscene`.

### Режим разработки (Dev)
Для запуска редактора из исходников:
1. Перейдите в папку `editor-app`
2. Установите зависимости: `npm install`
3. Запустите редактор: `npm run dev`

### Сборка (Build)
Для сборки готового приложения:
1. Перейдите в папку `editor-app`
2. Запустите нужный билд-скрипт, например: `npm run build:win` для Windows.
3. Готовый бинарный файл появится в папке `dist`.

## Интерфейс и Управление

Интерфейс редактора разделен на три основные зоны:
1. **Левая панель (Палитра нод и Действия)**: Здесь можно выбрать ноду для добавления на холст, а также выполнить отмену/возврат действий (Undo/Redo).
2. **Центральный холст (Canvas)**: Основная рабочая область, где располагаются ноды и связи между ними.
3. **Правая панель (Инспектор и Интеграция)**: Показывает параметры выбранной ноды/связи, а также содержит инструменты для загрузки GameMaker проекта и экспорта сцены.
4. **Дополнительные панели (Panels menu)**: Через верхнее меню можно отдельно показать `Logs / Warnings`, `Runtime JSON` и другие скрытые/floating панели.

Для `dialogue`-нод панель `Text` умеет показывать `Yarn Preview`: список нод (`title`) из выбранного `.yarn` файла и read-only содержимое выбранного блока. Для этого должен быть открыт GameMaker project, а у dialogue-ноды должен быть заполнен параметр `file`.

### Управление на холсте
* **Создание ноды**: Нажмите Среднюю Кнопку Мыши (СКМ/Колесико) по пустому месту холста. Будет создана нода того типа, который вы предварительно выбрали (или нода по умолчанию). Также можно добавлять ноды кликом из левой панели.
* **Перемещение холста (Pan)**: Удерживайте Правую Кнопку Мыши (ПКМ) и двигайте мышь.
* **Выделение рамкой (Box Select)**: Зажмите Левую Кнопку Мыши (ЛКМ) на пустом месте и потяните.
* **Выделение ноды/связи**: Клик ЛКМ по ноде или связи.
* **Удаление ноды/связи**: Клик ПКМ по ноде или связи.
* **Соединение нод**: Потяните ЛКМ от кружка (handle) одной ноды к кружку другой. Связи всегда идут слева направо (от `out` к `in`).
* **Настройка ожидания (Wait)**: Двойной клик по связи (ребру) открывает в инспекторе (справа) поле для ввода времени ожидания (в секундах) перед выполнением следующей ноды.
* **Условия на ребре (Edge Condition)**: В инспекторе выбранного ребра можно включить проверку `global`-переменной, задать поведение `skip` или `wait_until_true`, а также end-condition (`global_var`, `node_reached`, `timeout`).

### Preferences, доки и вспомогательные режимы
* **MiniMap**: Может быть включена или выключена через Preferences.
* **Zoom Speed**: Скорость зума холста настраивается через Preferences и влияет на wheel/trackpad zoom.
* **Dock Drop Preview**: В Preferences можно включить или отключить preview целевого места при перетаскивании панелей в dock.
* **Сворачивание dock-панелей**: Левый, правый и нижний dock можно сворачивать через узкие collapse bar-полоски, не теряя drag-and-drop hitbox.
* **Accent Color**: Акцентный цвет применяется к элементам editor UI, включая selection, hover и часть dropdown-состояний.
* **Canvas Background Image**: В Preferences можно выбрать собственную картинку для фона холста. Файл копируется во внутреннюю папку приложения (`userData`), поэтому editor не зависит от исходного пути. Поддерживаются режимы `Stretch` и `Cover (crop)`.
* **Auto-save**: В Preferences можно включить/отключить autosave и настроить интервал. Для уже сохранённой сцены autosave пишет в основной `.usc.json` и держит рядом autosave-backup файлы. Для новой, ещё не сохранённой сцены создаётся autosave-draft в `userData`.
* **Parallel branch ports**: Для `parallel_start` / `parallel_join` есть два режима:
  * `Shared single port` — визуально один общий порт, а ветки назначаются автоматически по порядку подключений.
  * `Separate ports per branch` — отдельный порт на каждую ветку, с ограничением: один connection на один branch-port.
* **Keyboard Shortcuts**: В Preferences показывается актуальный список shortcuts. Сейчас поддерживаются, например, `Ctrl+Shift+I` (Inspector), `Ctrl+1/2/3` (focus dock), `Space` (fit view), а для Zen mode используется `F11` в Dev и `F12` в production.
* **About modal**: В `Help -> About` открывается отдельное окно с версией приложения и ссылкой на эту документацию.
* **Advanced / Help**: В `Help` доступны дополнительные действия: `Open DevTools` и `Disable Hardware Acceleration`. Флаг hardware acceleration сохраняется в Preferences и применяется на следующем запуске приложения.
* **Language placeholder**: В Preferences уже есть поле `Language` (`English` / `Русский`). Пока оно остаётся disabled и служит подготовкой для будущего i18n-слоя, но значение уже хранится в `preferences.json`.
* **Logging + share**: Приложение пишет `console.log / warn / error` в файл `undefscene.log` внутри `userData` с простой ротацией. В `Help` доступно действие `Copy Log to Clipboard`, чтобы быстро отправить лог при баг-репорте.

## Интеграция с движком GameMaker

Редактор тесно интегрирован с движком игры для обеспечения автокомплита и валидации.

### Открытие проекта (Open Project)
В правой панели в блоке "Project Integration" нажмите **Open Project** и выберите файл `.yyp` вашего проекта Undefinedtale-888.
После успешной загрузки редактор просканирует ресурсы игры и настройки.

### Возможности интеграции
* **Автокомплит**: При редактировании параметров нод (например, имени объекта, спрайта, звука, комнаты или диалоговой ветки Yarn) редактор будет предлагать существующие ресурсы из загруженного проекта.
* **Синхронизация настроек**: Редактор читает файл `datafiles/cutscene_engine_settings.json` из проекта, чтобы получить настройки по умолчанию (например, дефолтный FPS, списки разрешенных функций и условий).
* **Поддержка Yarn**: Редактор сканирует файлы `.yarn` в папке `datafiles` и предлагает названия нод диалогов для автокомплита.

## Базовый Workflow

Рекомендуемый рабочий процесс при создании новой катсцены:

1. Откройте проект GameMaker через **Open Project (.yyp)**, чтобы editor получил ресурсы, `.yarn` файлы и настройки движка.
2. Создайте новую сцену через **New Scene** или загрузите шаблон через **Create Example**.
3. Добавьте основные ноды на холст и соедините их в нужном порядке.
4. Настройте параметры выбранной ноды в **Inspector**.
5. Если для перехода нужен таймер, задайте `wait` на ребре двойным кликом по линии.
6. Если для перехода нужны условия, настройте **Edge Condition** у выбранного ребра.
7. Проверяйте панель **Logs / Warnings**, чтобы ловить ошибки структуры графа и пустые поля.
8. Для `dialogue`-нод используйте **Text panel** и **Yarn Preview**, чтобы быстро сверять выбранный `.yarn` файл и `title` нужной ноды.
9. Сохраняйте рабочую сцену как `.usc.json`, а финальную версию для игры экспортируйте как engine `.json`.

## Импорт, Экспорт и Форматы

Редактор работает с двумя форматами файлов:

1. **Формат редактора (`.usc.json`)**: Это внутренний формат редактора, содержащий не только логику катсцены, но и визуальные данные (позиции нод на холсте, цвета и т.д.). **Этот файл нужен для сохранения прогресса работы в редакторе.**
2. **Формат движка (`.json`)**: Это чистый экспорт катсцены, который "понимает" движок GameMaker (объект `obj_cutscene_manager`). В нём нет визуальной информации, только структура команд (actions).

### Сохранение и Загрузка сцены (Рабочий процесс)
* **Save Scene / Save As...**: Сохраняет текущую сцену со всеми визуальными данными в файл `.usc.json`.
* **New Scene**: Создаёт пустую сцену с минимальным стартовым графом, чтобы можно было начать с чистого листа.
* **Create Example**: Загружает демонстрационную сцену с готовым графом для быстрого знакомства с редактором.
* **Open Scene**: Открывает ранее сохраненный файл `.usc.json` (или `.json`) в редакторе.

*Примечание: Редактор умеет открывать и engine `.json` файл. При таком reverse-import редактор восстанавливает editor graph из exported actions, включая линейные сцены, `wait`, `guard_global`/edge-condition, `parallel` и `branch`. Если структура JSON повреждена или реально неоднозначна, импорт может быть отклонён как unsupported format, чтобы редактор не строил неверный граф.*

### Отладочные панели
* **Logs / Warnings**: Показывает результаты валидации графа, разделённые по категориям `Errors`, `Warnings` и `Tips`.
* **Runtime JSON**: Теперь это отдельная hidden/floating panel, которую можно открыть через `Help` / advanced actions. Она показывает raw editor-state (`runtime.json`-style данные) и удобна для debugging или второго монитора.

### Экспорт для игры (Export)
* **Export for Engine**: Генерирует чистый `.json` файл с массивом `actions`, готовый для использования в игре.
Сохраняйте этот файл напрямую в папку `datafiles` вашего проекта GameMaker.

## Ручное редактирование

Хотя редактор предоставляет удобный интерфейс, вы по-прежнему можете редактировать `.json` файлы катсцен вручную в любом текстовом редакторе (например, в VS Code), если это необходимо.
Движок ожидает массив объектов (actions), где каждый объект имеет тип (`type`) и параметры. Подробнее о структуре команд движка читайте в разделе [API Катсцен](api.md).

## Поддерживаемые ноды (Типы команд)

Редактор поддерживает все основные команды движка:

* `start` / `end`: Начало и конец катсцены (технические ноды).
* `actor_create` / `actor_destroy`: Управление актерами (создание, удаление).
* `move`: Перемещение актера в точку.
* `follow_path`: Движение актера по пути (Path).
* `animate`: Воспроизведение анимации (спрайта) актером.
* `dialogue`: Запуск диалога (Yarn Spinner).
* `camera_track` / `camera_pan`: Управление камерой.
* `set_position`, `set_depth`, `set_facing`: Моментальное изменение свойств актера.
* `run_function`: Вызов кастомной GML функции (белый список в `cutscene_engine_settings.json`).
* `parallel_start`: Запуск нескольких веток действий параллельно.
* `branch`: Ветвление логики по условию.

Более подробную информацию о параметрах каждой команды смотрите в разделе [API Катсцен](api.md).

### Краткая шпаргалка по полям нод

Ниже перечислены самые частые поля, которые вы будете заполнять в Inspector:

* **`dialogue`**: `file`, `node`.
* **`move`**: `target`, `x`, `y`, `speed_px_sec`, `collision`.
* **`follow_path`**: `target`, `path`, `speed_px_sec`, `loop`.
* **`actor_create`**: `key`, `sprite_or_object`, `copy_from`, `x`, `y`.
* **`actor_destroy`**: `target`.
* **`animate`**: `target`, `sprite`, `loop`, `fps`.
* **`camera_track`**: `target`.
* **`camera_pan`**: `x`, `y`, `seconds`.
* **`camera_shake`**: `seconds`, `magnitude`.
* **`set_position`**: `target`, `x`, `y`.
* **`set_depth`**: `target`, `depth`.
* **`set_facing`**: `target`, `direction`.
* **`auto_facing`**: `target`, `enabled`.
* **`auto_walk`**: `target`, `enabled`.
* **`run_function`**: `function`, `args`.
* **`branch`**: `condition`.
* **`parallel_start` / `parallel_join`**: ветки управляются в editor UI через branch ports, а не через ручной ввод сырых полей.

### FAQ

**Почему editor сохраняет `.usc.json`, а игра использует `.json`?**  
`.usc.json` нужен для продолжения работы в редакторе: там есть позиции нод, panel state и другие editor-only данные. Обычный `.json` — это чистый экспорт для движка.

**Можно ли открыть обратно уже экспортированный engine `.json`?**  
Да. Редактор умеет reverse-import и пытается восстановить graph из массива `actions`.

**Почему `Runtime JSON` открывается не в обычных Panels?**  
Это advanced/debug panel. Она открывается через `Help`, чтобы не мешать основному набору рабочих панелей.

**Что делать, если в Logs есть warning, но экспорт всё равно доступен?**  
`warning` не блокирует экспорт, но почти всегда означает, что стоит проверить пустые поля, имена ресурсов, `dialogue` file/node или условия на ребре.

**Где искать логи приложения для баг-репорта?**  
Используйте `Help -> Copy Log to Clipboard` или файл `undefscene.log` в `userData`.
