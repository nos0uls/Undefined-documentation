# Структура Проекта (Project Structure)

Этот документ описывает основные папки и ключевые объекты проекта **Undefinedtale-888**.

## Основные Папки (Folders)

### 1. Объекты (Objects)
*   **Системы (Systems)**: Здесь лежат самые важные объекты, управляющие игрой.
    *   `obj_Init`: Точка входа в игру. Инициализирует глобальные переменные и загружает настройки.
    *   `obj_globalManager`: Главный менеджер. Отвечает за музыку, глобальные переходы, стек UI и общие события.
    *   `obj_camera`: Управляет камерой (следование за игроком, катсцены).
*   **Игровые объекты**:
    *   `obj_player`: Объект игрока.
    *   `obj_actor`: Базовый объект для NPC и персонажей катсцен.

### 2. Скрипты (Scripts)
Скрипты сгруппированы по функционалу.
*   `scr_init`: Скрипты инициализации.
*   `scr_player_...`: Логика игрока (движение, анимация).
*   `scr_ui_...`: Функции интерфейса (стек UI, проверка блокировки).
*   `scr_cutscene_...`: Система катсцен.
*   `scr_interaction_...`: Система взаимодействия.

### 3. Комнаты (Rooms)
*   `rm_init`: Первая комната при запуске (пустая, только для загрузки).
*   `rm_roomMenu`: Главное меню.
*   `rm_school_...`: Игровые локации (школа).

## Ключевые Объекты (Key Objects)

### `obj_Init`
Создается один раз в самом начале.
*   Загружает `global.player_settings`.
*   Загружает базовые глобальные структуры и настройки.
*   Собирает `global.input_map`.
*   Создает `obj_globalManager` как runtime-менеджер.

### `obj_globalManager`
Существует всегда (Persistent).
*   **Музыка**: Хранит текущий трек, управляет плавными переходами (`play_music`).
*   **UI Blocking**: Проверяет `scr_checkUIBlocking()` — блокирует ввод игрока при открытых меню, диалогах, настройках, сейвах и катсценах.
*   **Комнаты**: Определяет, является ли комната "меню" или "игрой".

### `obj_pointMarker`
Невидимый объект, привязанный к игроку. Показывает точку, куда игрок "смотрит" и с чем может взаимодействовать (нажать Z).
