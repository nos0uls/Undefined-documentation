# План фичи: игрок как «кукла» в катсценах (вариант B) + оптимизация движка катсцен

## Краткое резюме
Цель: в катсценах использовать **визуальную «куклу» (obj_actor)** вместо прямого управления спрайтом `obj_player`, при этом **реальный `obj_player` остаётся в комнате** и продолжает участвовать в коллизиях/триггерах/смене комнат.

Параллельно: убрать очевидные источники крашей и привести поведение к более последовательной модели (типобезопасные target_ref, предсказуемый resolve_target, снижение лишних проверок/строковых операций в Step).

---

## 1) Уточнение требований
### Что уже решено
- Выбран подход **B**: `obj_player` живой, визуал управляется отдельной «куклой».

### Вопросы (данных недостаточно)
1) Должен ли игрок быть **видимым** во время катсцены?
- Решено: игрок скрывается (`visible=false`), видна только кукла.

2) Должны ли интеракции/маркер (`obj_pointMarker`) работать во время катсцены?
- Решено: маркер **не скрываем**, он доступен только в дебаге.

3) Нужна ли возможность адресовать **реального игрока** из катсцен?
- Вариант по умолчанию: да, через отдельный ключ (например, `"player_body"`).

4) Производительность:
- Цель: стабильные 60 FPS. Если есть слабые платформы (старые ПК/ноуты), скажи.

Критерии готовности:
- Катсцена может двигать “игрока” и при этом корректно работают коллизии, roomchanger и прочие проверки по `obj_player`.
- Нет дубликатов игрока на экране.
- Нет падений из-за `instance_exists()`/чтения несуществующих переменных.

## 2) Анализ контекста проекта (что сейчас важно)
- `obj_cutsceneManager.start_cutscene()` выставляет `global.cutscene_active=true` и ставит игроку `can_move=false`.
- При этом `obj_player.Step` продолжает выполняться, но:
  - `scr_player_animation` и `scr_player_facing` выходят в катсцене (не меняют спрайт/направление).
  - `scr_player_marker_update` продолжает работать (если маркер есть и `facing_direction` валиден).
- `resolve_target("player")` сейчас возвращает реального игрока (`global.obj_player` или `instance_find(obj_player,0)`).

Вывод для варианта B:
- Чтобы визуал был “актерный”, нужно перенести управление спрайтом/анимацией на куклу.
- Чтобы коллизии/триггеры продолжали работать — реальный `obj_player` должен **физически перемещаться** вслед за куклой (или наоборот), но оставаться инстансом `obj_player`.

## 3) Архитектурный дизайн
### Основная идея
- На старте катсцены создаём `player_puppet` (объект `obj_actor`) в позиции игрока, копируем базовые визуальные параметры.
- `obj_player` скрываем (и опционально скрываем `marker_id`).
- Все команды катсцен, которые нацелены на `"player"`, направляем на куклу.
- Каждый Step катсцены синхронизируем физическое положение `obj_player` = положение куклы (так roomchanger/коллизии/проверки по obj_player видят правильную позицию).
- На завершении катсцены — уничтожаем куклу, возвращаем видимость игрока (и маркера).

### Данные/переменные (точные имена)
В `objects/obj_cutsceneManager/Create_0.gml`:
- `player_body = noone;` // реальный игрок
- `player_puppet = noone;` // кукла (obj_actor)
- `prev_player_visible = undefined;`
- `prev_marker_visible = undefined;`

Методы менеджера:
- `enable_player_puppet()`
- `disable_player_puppet()`

Правка `resolve_target(_ref)`:
- Если `_ref == "player"` и `instance_exists(player_puppet)` → возвращать куклу.
- Если `_ref == "player_body"` → возвращать реального игрока.

Синхронизация в `obj_cutsceneManager/Step_0.gml`:
- Если `instance_exists(player_body)` и `instance_exists(player_puppet)`:
  - `player_body.x = player_puppet.x;`
  - `player_body.y = player_puppet.y;`

Примечание: синхронизацию лучше делать **каждый Step менеджера**, чтобы не добавлять код в `obj_player`.

## 5) Риски
- Риск десинхронизации (кукла уехала, игрок не синхронизировался) → лечится жёсткой синхронизацией каждый Step.
- Риск, что где-то в игре ожидают видимого `obj_player` → решается возвратом `visible` после катсцены и возможностью отключать скрытие.
- Риск, что какие-то катсцены напрямую передают instance id игрока (`_pl`) в экшены → такие экшены будут двигать реальный `obj_player`, а не куклу. Нужно договориться о каноне: для катсцен использовать `"player"`.
