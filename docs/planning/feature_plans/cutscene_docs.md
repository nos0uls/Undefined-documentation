# План фичи: документация по катсценам (docs/)
 
 ## Краткое резюме
 Добавить папку `docs/` для фиксации канона катсцен-движка (цели как instance id, скорость в px/frame), модели актёров (`obj_actor` + `obj_dummy`) и правил интеграции с игроком. Это документация и не требует модификации исходников Chatterbox/Scribble.
 
 ## 1) Уточнение требований
 - **Область**: только документация катсцен-движка.
 - **Производительность**: не применимо (docs-only).
 - **Точки интеграции**:
   - `scripts/c_*`, `scripts/cutscene_*`
   - `scripts/scr_cutscene_classes/scr_cutscene_classes.gml`
   - `objects/obj_cutsceneManager/*`
 - **Критерии готовности**:
   - Новому участнику команды не нужно угадывать units/типы target.
   - Правила актёров/игрока явно описаны.
 
 ## 2) Анализ контекста проекта
 Канонические переменные runtime:
 - `global.active_cutscene_manager`
 - `global.active_cutscene_id`
 
 Персистентность:
 - `game_state.dat` (через `scr_game_state_*`) хранит общий game state и не должен хранить runtime-флаги катсцен.
 
 ## 3) Архитектурный дизайн
 ### Структура docs
 - `docs/cutscenes/overview.md`
 - `docs/cutscenes/api.md`
 - `docs/cutscenes/actors.md`
 - `docs/cutscenes/player_in_cutscene.md`
 
 ### Темы/канон (что фиксируем)
 - `target_ref`: instance id
 - единицы скорости: px/frame
 - объекты актёров: obj_actor + obj_dummy
 
 ## 4) План внедрения
 1. **Phase 1**: создать `docs/` и базовые markdown-файлы.
 2. **Phase 2**: заполнить раздел API точным списком обёрток, которые реально используются.
 3. **Phase 3**: добавить примеры (минимальные катсцены).
 4. **Phase 4**: поддерживать docs актуальными при изменении движка.
 
 ## 5) Риски
 - Риск низкий; документация не влияет на runtime.
 - Возможный риск: docs устареют при изменении API.
 
 ## 6) Альтернативы
 | Подход | Производительность | Сложность | Поддерживаемость |
 |---|---:|---:|---:|
 | A: `docs/` в корне репо | н/п | Низкая | Высокая |
 | B: только `project_docs/` | н/п | Низкая | Средняя |
 
 Рекомендация: A, т.к. папка `docs/` проще обнаруживается и ближе к коду.
