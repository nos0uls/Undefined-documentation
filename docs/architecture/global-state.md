# Глобальное Состояние (Global State)

Управление состоянием игры разделено на два уровня: **Runtime** (текущая сессия) и **Persistent** (сохраняемые данные).

## Runtime State (`obj_globalManager`)

`obj_globalManager` — это центральный нерв игры во время работы. Он существует в единственном экземпляре и управляет системами, которые должны работать непрерывно между комнатами.

### Ответственность
*   **Музыка**: Хранит текущий трек и управляет кросс-фейдом (`global.play_music`).
*   **UI Stack**: Отслеживает открытые окна для блокировки ввода.
*   **Debug**: Обрабатывает глобальные хоткеи (F1-F12).
*   **Уведомления**: Рисует всплывающие сообщения (`global.show_notification`).

## Persistent State (`game_state.dat`)

Долгосрочные данные, которые должны пережить перезапуск игры, хранятся в структуре `global.game_state` и сериализуются в файл.

### Структура данных
По умолчанию состояние выглядит так (`scr_game_state_default`):
```json
{
    "last_played_save_slot": "save1"
}
```

### API
Работа с состоянием ведется через скрипты `scr_game_state_*`:

*   **`scr_game_state_load()`**: Читает `game_state.dat` (формат `key=value`).
*   **`scr_game_state_save(state)`**: Записывает текущее состояние на диск.

!!! warning "Отличие от Save File"
    `game_state.dat` — это мета-данные **игры** (какой слот последний, глобальные настройки).
    Прогресс игрока (уровень, инвентарь) хранится в отдельных файлах слотов (`save1.txt`, `save2.txt`).

## Настройки Игрока (`player_settings.dat`)

Настройки управления, звука и графики вынесены в отдельную структуру `global.player_settings`.

*   Загрузка: `scr_loadSettings()`
*   Применение: `scr_applySettings()`
*   Сохранение: `scr_saveSettings()`

Это позволяет сбросить прогресс игры, не теряя настройки управления.
